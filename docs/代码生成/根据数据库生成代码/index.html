根据数据库生成代码

代码生成​

tip

日常开发碰到接口curd需要使用不同的对象，插件就是为了解决这个问题，当然还有生成根据模板生成curd完整代码的功能

代码模板文件在IDEA中存放的位置​

  • 代码模板文件默认放到：Scratches and Consoles/Extensions （中文：临时文件和控制台/扩展/com.una.database.database.generator）
  • 同时支持以下模板路径：${project.dir}/.idea/generator/templates 和 ${project.dir}/generator/templates

插件展示​

info
  1. 自动生成代码插件模板可以新增也可以使用内置的模板，新增模板位于项目根目录generator/templates/目录下
  2. 支持自定义文件后缀为了方便批量生成request save update delete等对象，模板顶部设置${gen.setType("custom.Save")}格式必须是custom.xxx， 当然xxx为文件后缀，可以参考velocity-custom.java目录下的文件例子
  3. 带有custom.xxx格式的都会使用Custom Setting的包路径，其他的还支持 controller entity xml dao service 对应xxx Setting
